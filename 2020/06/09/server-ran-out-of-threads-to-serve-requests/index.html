Server ran out of threads to serve requests.

發表於 2020-06-09

當在短暫的時間內對 Server 發送的 Request 過多時有可能會發生
Server ran out of threads to serve requests. Consider raising the ThreadsPerChild setting
的情況

這是在 Apache 的 Error Log 下所查到的訊息
代表著目前 Apache Server 所開的線程不足以應付如此大量的請求，因而造成 Server Crash

若在 httpd.conf 裡未設定 ThreadsPerChild 的話 Apache 會以預設開啟 64 個線程以供使用
如果一次性或是短暫時間內的 Request 超過就會 Crash 了
如有需要針對此做設定只需要於 httpd.conf 裡加上

如此即是將線程設定為 512 個以供使用，如有需要變更即只要變更此數字即可

Apache
CocoaPods
Ubuntu 14.04 安裝 LEMP
