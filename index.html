Homebrew 是什麼？

homebrew 是在 OS X 不可缺少的套件管理器，與 Linux 下的 apt-get 或 yum

安裝 Homebrew

打開終端機將下面的指令貼上即可：
/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
安裝完後即可使用 brew search/install/uninstall/update FORMULA 來做操作

安裝 Nginx

brew install nginx

nginx -t 測試 Nginx 設定有無錯誤

nginx -s reload|reopen|stop|quit 重新加載配置｜重新啟用｜停止｜退出 Nginx

Server ran out of threads to serve requests.

發表於 2020-06-09 | 分類於 Env

當在短暫的時間內對 Server 發送的 Request 過多時有可能會發生
Server ran out of threads to serve requests. Consider raising the ThreadsPerChild setting
的情況

這是在 Apache 的 Error Log 下所查到的訊息
代表著目前 Apache Server 所開的線程不足以應付如此大量的請求，因而造成 Server Crash

若在 httpd.conf 裡未設定 ThreadsPerChild 的話 Apache 會以預設開啟 64 個線程以供使用
如果一次性或是短暫時間內的 Request 超過就會 Crash 了
如有需要針對此做設定只需要於 httpd.conf 裡加上

如此即是將線程設定為 512 個以供使用，如有需要變更即只要變更此數字即可

CocoaPods

發表於 2020-06-09 | 分類於 code

在開發 iOS App 如果該 App 有需要用到第三方套件時原本我們都是如何處理？
先在網路上(Github)上找可用的第三方套件然後下載下來解壓縮後將該套件所有的程式碼皆拉進 Project 裡
或許會建一個目錄專門放置這些第三方套件，但之後如果所使用的套件有更新或是異動時我們還要先去確認現在所使用的版本跟異動後的版本是否有什麼影響等等
等同於是在這個專案裡需要控管自己的程式與這些第三方套件，且之後有需要更新一樣是要先至網路上將新版的下載後解壓再拉進 Project 裡
這些操作是麻煩的，若是有什麼地方操作錯誤那麼就整個 GG 了
所以在網路上看到有許多人推薦使用 CocoaPods 來做這些第三方套件的版本控管
透過 CocoaPods 我們只需要定義所需要使用的是那幾個第三方套件及其版本
之後安裝及更新等皆只需要一個指令 CocoaPods 都幫我們處理好了，真是簡單方便丫

CocoaPods 是一個 Ruby 的 Gem，所以只需要在 Terminal 下安裝指令
[sudo] gem install cocoapods
之後
pod setup
如此即完成 CocoaPods 的安裝

查詢 CocoaPods 版本可利用 pod --version
查詢套件可利用 pod search + 'Github 上的套件名稱
如：查詢 AFNetworking => pod search AFNetworking

等在套件名稱內有與 AFNetworking 相關的皆會條列出來

那麼要如何將 CocoaPods 引用在 Xcode 中呢？
首先需要於 Project 目錄中建立 Podfile : vi Podfile

如此即是設定此 Project 是專為 iOS 7.0 以上所開發的
且使用了以下六個第三方的套件
AFNetworking 2.5.4
objective-zip 0.8.3
FMDB 2.5
MBProgressHUD 0.9.1
Reachability 3.2
SSKeychain 1.2.3

在輸入完 Podfile 後即利用 pod install 來讓 CocoaPods 安裝這幾個套件

如此即完成這些套件的安裝
之後皆要利用 projectname.xcworkspace 來開啟 Xcode
如此才能夠將 CocoaPods 所管理的套件引用在 Project 裡

如果利用 projectname.xcworkspace 開啟 Xcode 後有 Error 如 ： _OBJC_CLASS_$_AFHTTPRequestOperation", referenced from:
等情況可先於 Prject -> Build Settings 查詢 “Other Linker Flags” 如果看到這個是以粗體顯示的情況
則點選後按下 delete 鍵會將原本的資料回復，而 “Other Linker Flags” 應該會恢復為原本的字體
如果這樣操作還是無法解決，則可能需要將原本所做的事皆復原後重新安裝 Pods

網頁設置 IE 兼容模式

發表於 2020-06-09 | 分類於 code

網頁設置要處理 IE 的相容性，否則有些網頁利用 IE 開啟時會因為使用的 engine 不同導致網頁的呈現完全跑掉
要如何設置 IE 用什麼樣的 engine 來執行網頁呢？
只需要於頁面中增加 <meta http-equiv="X-UA-Compatible" content="IE=?" /> 即可

如果要強制瀏覽器呈現特定版本的標準，如不支援 IE7 以下 :
<meta http-equiv="X-UA-Compatible" content="IE=11; IE=10; IE=9; IE=8; IE=7" />
上面的設定即只支援 IE11 ~ IE7 ， IE6 及以下皆不支援

當然也可以指定只支援某一個 IE 版本，如下
<meta http-equiv="X-UA-Compatible" content="IE=8" />
這即代表只支援 IE8

如果以使用者環境安裝的版本為主的話
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
即什麼版本的 IE 即用其版本的標準模式

Mac 設定 Apache 開啟 User Level Root

發表於 2020-06-09 | 分類於 Env

首先於家目錄中建立 Sites 目錄
mkdir ~/Sites

接下來於 /etc/apache2/users/ 下建立 username.conf
cd /etc/apache2/users/
sudo vi username.conf
並將名字帶入下面的 username 裡

再來將這個設定檔更改權限
sudo chmod 644 username.conf
之後於 httpd.conf 中開啟一些設定
sudo vi /etc/apache2/httpd.conf
將以下三個模組打開
LoadModule authz_core_module libexec/apache2/mod_authz_core.so
LoadModule authz_host_module libexec/apache2/mod_authz_host.so
LoadModule userdir_module libexec/apache2/mod_userdir.so
最後打開 incloude userdir
Include /private/etc/apache2/extra/httpd-userdir.conf
接著修改 httpd-userdir.conf
sudo vi /etc/apache2/extra/httpd-userdir.conf
將引用 users 下設定檔的註解拿掉
Include /private/etc/apache2/users/*.conf
重新啟動 Apache
sudo apachectl restart
之後即可於瀏覽器中 http://localhost/~username/ 看到正常的網頁了
